# Mock数据测试指南

在没有后端服务的情况下，本项目提供了完整的Mock数据模拟方案，可以测试前端所有页面和功能。

## 启用Mock数据

Mock数据功能已经默认启用，配置位于 `src/api/axiosInstance.ts` 文件中的：

```typescript
// 配置是否使用mock数据
const USE_MOCK = true // 设为true使用mock数据，false使用真实API
```

需要使用真实后端API时，将 `USE_MOCK` 设为 `false` 即可。

## Mock数据结构

Mock数据定义在 `src/api/mockService.ts` 文件中，包含以下数据：

### 1. 用户数据
- 模拟用户信息（学号、姓名、邮箱等）
- 登录/注册/验证码等认证流程模拟

### 2. 社团数据
- 4个推荐社团（科技创新社、文学社、篮球俱乐部、摄影协会）
- 社团详情、成员数、活动等信息

### 3. 活动数据
- 5个模拟活动，包括编程大赛、文学沙龙、篮球比赛等
- 活动时间、地点、参与人数等详细信息

### 4. 统计数据
- 社团总数、活动总数、活跃用户数等
- 图表数据（月度活动数量、新增成员等）

## 测试步骤

1. 启动开发服务器：
   ```
   npm run dev
   ```

2. 访问前端页面（默认地址：http://localhost:3000 或 http://localhost:3001）

3. 测试登录功能：
   - 账号密码登录：可以输入任意账号密码（mock会自动通过）
   - 验证码登录：输入任意手机号，点击发送验证码，然后输入任意验证码
   - 注册功能：填写表单后可以完成注册模拟

4. 测试首页功能：
   - 登录后可以查看统计数据
   - 浏览推荐社团列表
   - 查看近期活动
   - 查看数据图表

5. 测试社团和活动交互：
   - 加入/退出社团
   - 报名/取消活动
   - 查看社团详情

## 自定义Mock数据

如需修改Mock数据，可以编辑 `src/api/mockService.ts` 文件中的模拟数据对象：

- `mockUser`: 用户信息
- `mockClubs`: 社团列表
- `mockActivities`: 活动列表
- `mockStats`: 统计数据

## 注意事项

1. Mock数据是完全离线的，不需要网络连接
2. 所有API调用会被拦截并返回模拟数据
3. 操作（如加入社团、报名活动）不会持久保存，刷新页面后会重置
4. 如需连接真实后端，记得将 `USE_MOCK` 设为 `false`

## 开发建议

在前端开发阶段，建议先使用Mock数据进行开发和测试，确保UI和交互逻辑正确后，再连接真实后端服务进行集成测试。
